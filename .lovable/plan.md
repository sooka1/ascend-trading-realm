## الهدف
توليد ترجمات كاملة لجميع اللغات الـ35 الموجودة في `LANGUAGES` باستخدام قاموس الإنجليزية (`DICTS.en`) كمصدر، بحيث يحتوي كل قاموس على كامل المفاتيح (~500 مفتاح) بدلًا من الاعتماد على fallback إلى الإنجليزية.

## المقاربة

بدل كتابة ~17,500 سطر ترجمة يدويًا (غير عملي)، سأستخدم **Lovable AI Gateway** لتوليد الترجمات دفعة واحدة عبر سكربت مؤقت، ثم حفظ النتائج كملفات JSON ثابتة يتم تحميلها في `i18n.tsx`.

### الخطوات

1. **فصل القواميس إلى ملفات JSON**
   - إنشاء `src/lib/locales/en.json` من `DICTS.en` الحالي (مصدر الحقيقة).
   - الحفاظ على `ar`, `fr`, `es`, `tr` كما هي في ملفات JSON مستقلة.

2. **سكربت توليد الترجمات** (`scripts/translate-locales.mjs`)
   - يقرأ `en.json` ويرسل المفاتيح دفعات إلى `google/gemini-2.5-flash` عبر Lovable AI Gateway.
   - Prompt يطلب ترجمة قيم JSON مع الحفاظ على placeholders مثل `{year}` و`{seconds}` وعلامات الترقيم.
   - يحفظ الناتج في `src/lib/locales/<code>.json` لكل لغة من الـ30 المتبقية.
   - يُشغَّل مرة واحدة يدويًا (`bun run scripts/translate-locales.mjs`) — ليس جزءًا من البناء.

3. **تحديث `src/lib/i18n.tsx`**
   - استيراد كل ملفات JSON عبر `import.meta.glob('./locales/*.json', { eager: true })`.
   - بناء `DICTS` ديناميكيًا من الملفات المستوردة.
   - الاحتفاظ بـ fallback إلى `en` كأمان.

4. **التحقق**
   - فتح المعاينة وتبديل عدة لغات (de, ja, ko, ru, hi, zh-cn, ku) للتأكد من ترجمة النصوص المرئية.

### تفاصيل تقنية

- **المفتاح**: `LOVABLE_API_KEY` متاح تلقائيًا عبر Lovable AI Gateway (لا حاجة لإضافة سر).
- **حجم الطلب**: ~30 لغة × ~500 مفتاح. سنُقسّم كل لغة إلى ~5 دفعات لتفادي حدود الـ context.
- **التكلفة**: `gemini-2.5-flash` مجاني حتى 13 أكتوبر 2025 — إن لم يكن كذلك سأستخدم موديل مدفوع صغير.
- **الحفاظ على placeholders**: prompt صريح + تحقق برمجي بعد كل استجابة (regex على `{...}`).
- **RTL**: اللغات RTL (ku, he, fa, ur) ستُترجم بشكل طبيعي؛ الاتجاه يُدار بالفعل عبر `LANGUAGES[].dir`.

### ما لن يتغير
- شكل مفاتيح الترجمة.
- منطق `useI18n()` / `t()`.
- LANGUAGES أو ENTRIES في `language-switcher`.

### تنبيه
الترجمات المولّدة بالذكاء الاصطناعي جيدة لكنها ليست بمستوى مترجم بشري محترف. يُنصح بمراجعتها لاحقًا للنصوص المالية الحساسة (تحذيرات المخاطر، الشروط).
